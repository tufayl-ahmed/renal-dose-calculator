## Needs Manual Review

- Warfarin: FDA labels emphasize closer INR monitoring and higher bleeding risk in renal dysfunction, but I did not add a JS rule because I did not find a clean adult renal dose adjustment or explicit no-adjustment statement in the reviewed label snippets.
- Procainamide: DailyMed labeling states renal excretion is the main elimination route and reduced renal excretion lowers the maintenance dose rate needed, but the label text found did not provide a CrCl/eGFR dosing table or numeric renal dose adjustment.
- Captopril: labeling recommends smaller or less frequent doses and slower titration in significant renal impairment, but the source text found did not provide a specific CrCl/eGFR dose table.
- Hydrochlorothiazide and chlorthalidone: labels include anuria contraindication and renal-risk cautions, but not a clean adult renal dosing adjustment table suitable for CrCl banding.
- Rivaroxaban/apixaban/dabigatran/edoxaban/enoxaparin: anticoagulant rules are highly indication-specific and affected by interacting drugs, weight, age, dialysis status, or bleeding risk. Current JS records preserve these caveats in `condition`, but they need pharmacist/clinician review before use in production.
- Digoxin: a draft record points to label maintenance tables rather than encoding every lean-body-weight/CrCl cell. Full table encoding should be a separate manual curation pass.
- Torsemide and metolazone: retained in JS only as renal-disease indication dosing/manual-review records. They are not clean renal impairment dose-adjustment rules.
- Sotalol: AFIB/AFL and ventricular arrhythmia interval rows are both captured, but the record remains high-risk and needs specialist review because initiation/reinitiation requires ECG/QT monitoring.
- Eptifibatide: dialysis dependency is represented as a contraindication branch in `condition` text because the current rule schema lacks a dialysis-modality field.

## Curation Caveats

- All records are `confidence: "draft-source-extracted"` only.
- Several records use source statements such as "no renal dose adjustment" or "not recommended/contraindicated" where the label is explicit.
- For drugs where the label uses eGFR instead of CrCl, the rule text preserves eGFR in `condition`; the app should avoid treating these as interchangeable without a UI note.
- I used DailyMed FDA-label pages or DailyMed lookup pages only for the records in this draft.
- Dialysis, CAPD, and indication-dose distinctions are currently encoded in condition text, not structured fields.
